# 修复语法高亮样式冲突

根据之前的搜索结果，问题的主要原因在于 `highlight.js` 的样式文件与 Tailwind CSS 的配置不匹配。具体来说：
1.  **样式冲突**：`src/main.js` 引入了 `github.css`（浅色主题），但 `src/views/Post.vue` 和 `src/views/Editor.vue` 引入了 `atom-one-dark.css`（深色主题）。同时，`tailwind.config.js` 可能对 `prose` 插件进行了配置，导致代码块背景色与高亮主题不协调。
2.  **重复引入**：在多个组件中重复引入 `highlight.js` 的 CSS 文件可能会导致样式覆盖顺序问题。

## 修复计划

我将采取以下步骤彻底解决这个问题：

1.  **统一 CSS 引入**：
    *   在 `src/main.js` 中**保留并统一**引入 `atom-one-dark.css`，作为全局高亮主题。
    *   移除 `src/views/Post.vue` 和 `src/views/Editor.vue` 中局部的 CSS 引入，避免重复和冲突。

2.  **验证 Tailwind 配置**：
    *   检查 `tailwind.config.js`，确保 `prose` 插件的代码块样式没有覆盖 `highlight.js` 的颜色（特别是文本颜色）。通常 `prose` 会设置默认的 `color`，这可能会覆盖 `hljs` 的颜色。我们需要在 CSS 中强制 `hljs` 类名的样式优先级。

3.  **强制样式覆盖 (如果需要)**：
    *   在 `src/style.css` 中添加一段 CSS，确保 `.hljs` 及其子元素的颜色样式优先级高于 Tailwind Typography 插件的默认样式。

## 执行步骤

1.  **修改 `src/main.js`**：引入 `atom-one-dark.css`，移除 `github.css`。
2.  **修改 `src/views/Post.vue` & `src/views/Editor.vue`**：移除局部的 CSS import。
3.  **修改 `src/style.css`**：添加 `.prose pre code` 的样式重置，确保 highlight.js 生效。

这样可以确保全站统一使用 Atom One Dark 主题，且不会被 Tailwind 的 Typography 插件意外覆盖。
