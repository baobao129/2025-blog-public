# 管理后台 UI 修复与编辑器独立化计划

用户反馈管理后台的新建/编辑页面与列表页叠加显示，导致布局混乱和溢出。同时需要增加 Markdown 语法提示功能。

## 1. 问题分析与修复方案
- **问题**：目前的 `Admin.vue` 使用了 `fixed inset-0` 的模态框来显示编辑器，但这在某些屏幕尺寸或样式配置下可能导致 z-index 冲突或滚动条双重出现，造成"叠加"和"溢出"的视觉 bug。
- **解决方案**：彻底废弃模态框设计，将编辑器拆分为一个**独立的路由页面** (`/editor`)。
    - 点击"新建文章" -> 跳转至 `/editor`。
    - 点击"编辑" -> 跳转至 `/editor/filename.md`。
    - 这样可以保证编辑器拥有独立、干净的全屏空间，互不干扰。

## 2. 功能增强：Markdown 语法提示
- 在编辑器页面增加一个悬浮按钮或工具栏按钮 "Markdown 语法"。
- 点击后从右侧滑出抽屉 (Drawer) 或弹出模态框，显示常用的 Markdown 语法速查表（标题、列表、代码块、链接、图片、以及新增的 `<small>` 等 HTML 标签）。

## 3. 执行步骤
1.  **创建 `src/views/Editor.vue`**：将 `Admin.vue` 中的编辑器代码迁移至此，并进行布局优化（全屏 Flex 布局）。
2.  **更新 `src/router/index.js`**：注册 `/editor` (新建) 和 `/editor/:filename` (编辑) 路由。
3.  **重构 `src/views/Admin.vue`**：移除模态框代码，将"新建"和"编辑"按钮的行为改为路由跳转。
4.  **实现 Markdown 提示组件**：在 `Editor.vue` 中集成语法提示功能。

## 4. 视觉风格
保持 "Zen Minimalist" 风格：
- 编辑器页面采用极简设计，左侧文件设置，中间写作区。
- 语法提示侧边栏采用浅灰色背景，代码示例清晰易读。
