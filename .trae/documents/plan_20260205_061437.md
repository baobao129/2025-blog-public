# 文章排版美化与功能增强计划

用户希望文章页面不仅仅是"单独的 Markdown 文档"，而是经过精心美化的、支持丰富排版（如大小标题、`<small>` 标签）的阅读体验。

## 1. 排版美化 (Typography Polish)
目前的 Tailwind Typography 默认样式较为基础。我们将对其进行深度定制，使其符合 "Zen Minimalist" 的风格。
- **标题层级**：明确区分 H1-H4 的字号、粗细和间距，增加 H2/H3 的顶部间距以区分章节。
- **正文排版**：优化行高 (line-height) 和段间距 (margin-bottom)，提升阅读舒适度。
- **特殊标签支持**：显式支持 `<small>`, `<sub>`, `<sup>`, `<mark>` 等 HTML 标签的样式，确保它们在 Markdown 中渲染正常。
- **引用块 (Blockquote)**：设计更具装饰性的引用样式（如左侧粗线条 + 背景色）。

## 2. Markdown 渲染增强
- **配置 Marked**：确保 `marked` 允许渲染 HTML 标签（默认通常允许，但需要确认配置）。
- **代码高亮**：引入 `highlight.js` 或 `shiki` 为代码块添加语法高亮，这是技术博客的标配。

## 3. 文章结构优化
- **侧边目录 (TOC)**：解析文章标题生成目录，固定在右侧（桌面端），方便长文导航。
- **阅读进度条**：保留并优化现有的顶部进度条。

## 4. 执行步骤
1.  **增强 Markdown 解析**：检查 `marked` 配置，引入 `highlight.js` 实现代码高亮。
2.  **定制 Typography**：修改 `tailwind.config.js`，深度定制 `prose` 类的 CSS。
3.  **重构 Post.vue**：
    - 引入代码高亮 CSS。
    - 优化文章容器布局，为 TOC 预留空间（可选，视复杂度而定，优先保证正文美观）。
    - 增加对 HTML 标签的样式支持。

## 关键技术点
- **Tailwind Typography (`prose`)**：这是核心。我们将通过 `tailwind.config.js` 中的 `typography` 插件配置来覆盖默认样式。
- **CSS 变量**：继续沿用现有的颜色系统，确保风格统一。
