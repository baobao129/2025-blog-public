# 优化代码高亮与编辑器主题

根据您的需求，我将进一步优化文章阅读界面和编辑器的颜色显示，使其更接近您提供的图片风格（Atom One Dark 风格）。

## 1. 优化文章阅读界面 (Post.vue)

### 变更内容

* **更换高亮主题**：将之前引入的 `github-dark.css` 替换为 `atom-one-dark.css`。

* **原因**：`atom-one-dark` 的色彩更丰富、对比度更高，更符合您图片中展示的现代编辑器风格。

## 2. 优化编辑器界面 (MonacoEditor.vue)

### 变更内容

* **自定义深色主题**：不使用默认的 `vs-dark`，而是定义一个名为 `one-dark` 的自定义主题。

* **配色调整**：

  * 背景色设为 `#282c34` (Atom One Dark 经典背景)。

  * 调整 Markdown 语法的颜色（如标题、链接、引用等），使其更鲜艳。

* **功能说明**：

  * **编辑器限制**：Monaco Editor 在 Markdown 模式下，**无法**自动对代码块内部的代码（如 `vue ... `  ）进行语法高亮。这是编辑器的技术限制。

  * **折衷方案**：虽然编辑时代码块内部是纯色的，但在右侧的**语法助手**预览区或**文章发布后**的阅读界面，都能看到完美的高亮效果。

## 3. 验证

* 确认 `Post.vue` 中的代码块使用了 Atom One Dark 风格。

* 确认 `MonacoEditor.vue` 的背景和文字颜色变为 One Dark 风格。
