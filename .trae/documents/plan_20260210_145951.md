# 修复搜索框、背景切换和主题切换问题

根据用户反馈，当前存在三个主要问题：
1.  **右上角搜索框显示异常**：看起来样式崩坏，无法正常输入。
2.  **右下角背景修改按钮无法点击**：可能被遮挡或 `z-index` 问题。
3.  **主题切换按钮失效**：点击无反应。

## 1. 修复搜索框 (Home.vue)
**问题分析**：
- `GlassCard` 作为搜索框容器，可能其默认样式（padding, overflow 等）与内部 input 不兼容。
- 搜索框高度可能未正确设置，导致 input 被压缩。

**解决方案**：
- 调整搜索框容器的 CSS，确保 input 有足够的空间。
- 检查 `GlassCard` 的 `variant="light"` 样式，确保没有不必要的 margin/padding 干扰。
- 显式设置搜索框的高度和 input 的样式。

## 2. 修复背景修改按钮 (BackgroundManager.vue)
**问题分析**：
- 背景管理器 `BackgroundManager` 位于 `App.vue` 的最底层，但在新的双栏布局中，`Sidebar` 和 `Main` 内容区可能遮挡了右下角的固定按钮。
- `z-index` 层级可能不足，被 `Sidebar` 或 `Main` 的背景覆盖。

**解决方案**：
- 提高 `BackgroundManager` 中设置按钮的 `z-index`（例如 `z-[9999]`）。
- 确保按钮位于所有布局层之上。

## 3. 修复主题切换按钮 (ThemeToggle.vue)
**问题分析**：
- 主题切换按钮使用了 `vueuse` 的 `useDark`。
- 如果按钮点击无效，可能是因为：
    - `Sidebar` 的 `z-index` 或布局问题导致点击事件被拦截。
    - `useDark` 配置未生效（例如 Tailwind 的 `darkMode: 'class'` 未正确配置）。
    - 按钮本身的样式导致点击区域过小或被遮挡。

**解决方案**：
- 检查 `Sidebar.vue` 中 `ThemeToggle` 的位置和层级。
- 确保 Tailwind 配置中启用了 `darkMode: 'class'`。
- 检查 `ThemeToggle.vue` 组件逻辑。

## 4. 执行计划

1.  **修改 `Home.vue`**：优化搜索框样式，移除不必要的 `GlassCard` 包装或调整其样式，使其更像一个标准的输入框。
2.  **修改 `BackgroundManager.vue`**：提升设置按钮的 `z-index`，确保其在最顶层。
3.  **修改 `Sidebar.vue` / `ThemeToggle.vue`**：检查主题切换按钮的层级和点击事件，确保其可交互。
4.  **验证**：逐一测试三个功能是否恢复正常。
