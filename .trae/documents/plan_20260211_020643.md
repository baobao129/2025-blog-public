# 移除主题切换并彻底修复背景切换按钮

根据用户反馈，我将执行以下操作：

1.  **移除主题切换按钮**：从 `Sidebar.vue` 中彻底删除 `ThemeToggle` 组件及其引用。
2.  **彻底修复背景切换按钮**：
    *   **问题根源**：目前的 `BackgroundManager` 位于 `App.vue` 的模板结构中，可能被其他层（如 `Sidebar` 或 `Main` 内容区的背景）遮挡。即使设置了 `z-index: 9999`，如果父容器存在 `overflow: hidden` 或层叠上下文问题，依然可能导致点击穿透失效。
    *   **解决方案 (Portal/Teleport)**：将背景设置按钮和面板使用 Vue 的 `<Teleport to="body">` 直接挂载到 `<body>` 标签下。这样可以完全脱离当前的 DOM 层级限制，确保它永远位于最顶层且不受父级样式影响。

## 执行计划

1.  **修改 `Sidebar.vue`**：
    *   删除 `ThemeToggle` 组件的导入和使用。
    *   清理相关布局代码。

2.  **修改 `BackgroundManager.vue`**：
    *   使用 `<Teleport to="body">` 包裹设置按钮和面板。
    *   确保按钮拥有固定的屏幕定位 (`fixed`) 和最高的 `z-index`。
    *   添加调试用的边框或背景色（临时），确保按钮区域可见且可点击（如果需要）。

3.  **验证**：
    *   确认左侧栏不再显示主题切换按钮。
    *   确认右下角背景设置按钮可以点击，并正常弹出面板。
