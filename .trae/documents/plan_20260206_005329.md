# 修复 Frontmatter 显示、代码高亮及编辑器主题

根据您的反馈，我将进行以下修改：

## 1. 修复文章详情页 (Post.vue)

### 问题分析
- **Frontmatter 泄露**：目前直接渲染了整个 Markdown 文件，导致顶部的元数据（如 `cover: ...`）显示在正文中。
- **代码无高亮**：虽然配置了高亮逻辑，但未引入对应的 CSS 样式文件。

### 修改计划
- **引入 Frontmatter 解析**：使用 `src/utils/frontmatter.js` 中的 `parseFrontmatter` 函数，将元数据与正文分离，仅渲染正文内容。
- **引入高亮样式**：导入 `highlight.js/styles/github-dark.css`（或其他深色主题），确保代码块显示语法颜色。

## 2. 修改编辑器主题 (MonacoEditor.vue)

### 问题分析
- **编辑器颜色**：当前配置为 `theme: 'vs'` (浅色)，导致背景为白色。

### 修改计划
- **启用深色主题**：将 Monaco Editor 的主题配置改为 `vs-dark`，使其背景变为灰黑色，与 VS Code 内部体验一致。

## 3. 验证
- 确认文章详情页不再显示 `cover` 等元数据信息。
- 确认文章详情页的代码块有正确的语法高亮颜色。
- 确认编辑页面右侧编辑器背景变为深色。
