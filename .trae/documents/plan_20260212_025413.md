# 修复文章详情页代码高亮问题

## 问题分析
用户反馈文章详情页（`Post.vue`）中的代码块没有颜色高亮。
经过检查 `src/views/Post.vue`，发现 `marked` 的配置中缺少 `langPrefix` 选项。
就像之前修复 `Editor.vue` 一样，`highlight.js` 的 CSS 样式（`atom-one-dark.css`）依赖于 `.hljs` 类名。
默认情况下，`marked` 生成的代码块类名是 `language-css` 或 `language-js`，缺少 `hljs` 前缀，导致样式无法匹配。

## 修复方案
在 `src/views/Post.vue` 中更新 `marked.setOptions`，添加 `langPrefix: 'hljs language-'`。

## 执行计划
1.  修改 `src/views/Post.vue`。
2.  找到 `marked.setOptions` 配置块。
3.  添加 `langPrefix: 'hljs language-'`。

这应该能立即解决问题，因为 `highlight.js` 和 CSS 样式已经正确引入了。
