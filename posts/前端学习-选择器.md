# 选择器

选择器：帮助你精准的选中想要的元素

## 简单选择器

**1. ID选择器**
``` CSS
/* 为所有id为intro的元素设置字体加粗 */
#intro {
  font-weight:bold
}

<p id = "intro">This is a paragraph of introduction.</p>
```
**2. 元素选择器**
```CSS
/* 为所有段落文字设置蓝色 */
p {
color: blue;
}

<p>This is a paragraph of introduction.</p>
```
**3. 类选择器**
```CSS
/* 为所有段落文字设置红色 */
red {
color: red;
}

<p class = "red">This is a paragraph of introduction.</p>
```

**4. 通配符选择器**

*，选中所有的元素

```CSS
/* 为所有元素设置红色 */
* {
color: red;
}

<p>This is a paragraph of introduction.</p>
```
**5. 属性选择器**

根据属性名和

```CSS
/* 选中所有具有href属性的元素 */
[href]{
    color:red;

}
<a href = "https://www.sina.com">新浪</a>
<a href = "https://www.baidu.com">百度</a>
```
全部变红

```CSS
/* 选中具有href属性的元素，并且href的值为"https://www.sina.com" */
[href = "https://www.sina.com"]{
    color:red;
}
<a href = "https://www.sina.com">新浪</a>
<a href = "https://www.baidu.com">百度</a>
```
只有新浪变红

```CSS
[class ~= "b"]{
  color: red;
  font-weight: bold;
  display: block;
}

  <a class = "a b c d" href="https://www.baidu.com">百度</a>
  <a class = "a c d" href="https://www.jd.com">京东</a>
```
只会百度变红。

**[attr~=value]**
表示带有以attr命名的属性元素，并且该属性是一个以空格作为分隔的值列表，其中至少一个值匹配"value"。

>更多：https://developer.mozilla.org/zh-CN/docs/Web/CSS/Reference/Selectors/Attribute_selectors

**6. 伪类选择器**

选中某些元素的某种状态

  1. link：超链接未访问时的状态
  2. visited：超链接访问过后的状态

  ```CSS
  a:link {
    color: chocolate;
  }

  a:visited {
    color: red;
  }


    <a href="https://www.baidu.com">百度</a>
    <a href="https://www.jd.com">京东</a>

  ```

  3. hover：鼠标悬停状态

  ```CSS
  /* 选中鼠标悬停时的a元素 */
  a:hover{
    color:red;
  }

    <a href="https://www.baidu.com">百度</a>
    <a href="https://www.jd.com">京东</a>
  ```

  4. active： 激活状态，鼠标按下状态

  ```CSS
  /* 鼠标按下时的a元素 */
  a:hover{
    color:red;
  }

    <a href="https://www.baidu.com">百度</a>
    <a href="https://www.jd.com">京东</a>

  ```

<mark>**如果需要设置属性，需要按照这样的顺序进行书写**</mark>

爱恨法则：love hate


**7. 伪元素选择器**

生成并选中某个元素内部的第一个子元素或最后一个子元素

before：在元素中生成一个子元素，并且子元素一定是第一个元素。

after：在元素中生成一个子元素，并且子元素一定是最后一个元素。

```CSS

span::before {
  /* content，只有before和after中才能够使用，表示生成的子元素中的内容 */
  content: "《";
  color: red
}

span::after {
  contetn: "》";
}

  <p>
      国务院办公厅转发民政部等10部门<span>关于进一步加强养老机构安全管理的意见</span>。
  </p>
```

## 选择器的组合

1. 并且

    连续一起写，如`p.red { color:red }`，两个选择器连续写在一起。

2. 后代元素 —— 空格

```CSS

div.red li {
    color: #008c8c;
}

<div class = "red">
    Lorem.
    <ul>
        <li>Lorem.</li>
        <li>Harum.</li>
        <li>Eum!</li>
        <li>Impedit?</li>
    </ul>
</div>
```

3. 子元素 —— >

```CSS
/* div元素下的ul元素下的li元素 */
div>ul>li{
  color: #008c8c;
}
```

4. 相邻兄弟元素 —— +

```html
<style>
  .special {
    color: red;
  }

  /* 类选择器special的下一个li元素颜色为#008c8c */
  .special+li {
    color: #008c8c;
  }
<style>

<div>
  <ul>
    <li>Lorem!</li>
    <li class = "special">nostrum porro?</li>
    <li>dolore!</li>
    <li>rem!</li>
    <li>quo?</li>
  </ul>
</div>
```

5. 兄弟元素 —— ~

```html
<style>
  .special {
    color: red;
  }

  /* 类选择器special的后面所有li兄弟元素颜色为#008c8c */
  .special~li {
    color: #008c8c;
  }
<style>

<div>
  <ul>
    <li>Lorem!</li>
    <li class = "special">nostrum porro?</li>
    <li>dolore!</li>
    <li>rem!</li>
    <li>quo?</li>
  </ul>
</div>
```


## 选择器的并列

多个选择器，用逗号分隔

语法糖：指编程语言中添加的简化语法结构。 这类语法不改变语言功能，但能提升代码可读性并减少错误率。

# 更多选择器

## 更多伪类选择器

1. first-child

选择第一个子元素

```CSS
<style>
    /* 选择li元素，并且li元素必须是第一个子元素 */
    li:first-child{
      color: red;
    }
</style>

<body>
  <div>
    <ul>
      /* 第一个元素变红 */
      <li>Lorem.</li>
      <li>Expedita?</li>
      <li>Eveniet.</li>
      <li>Explicabo!</li>
      <li>Commodi.</li>
    </ul>
  </div>
</body>


```

```CSS
<style>
/* 选择li元素，并且li元素必须是第一个子元素 */
    li:first-child{
      color: red;
    }
</style>

<body>
  <div>
    <ul>
      <a href="www.baidu.com">百度</a>
      /* 第一个元素不变红，只选择第一个子元素并且是li元素才会变化 */
      <li>Lorem.</li>
      <li>Expedita?</li>
      <li>Eveniet.</li>
      <li>Explicabo!</li>
      <li>Commodi.</li>
    </ul>
  </div>
</body>

```

2. first-of-type

```CSS
<style>
    /* 选择子元素中第一个li元素 */
    li:first-of-type{
      color: red;
    }
</style>

<body>
  <div>
    <ul>
      <a href="www.baidu.com">百度</a>
      /* 第一个元素变红，选择第一个li元素使其变红 */
      <li>Lorem.</li>
      <li>Expedita?</li>
      <li>Eveniet.</li>
      <li>Explicabo!</li>
      <li>Commodi.</li>
    </ul>
  </div>
</body>


3. last-child

```CSS
<style>
/* 选择li元素，并且li元素必须是最后一个子元素 */
    li:last-child{
      color: red;
    }
</style>

<body>
  <div>
    <ul>
      <a href="www.baidu.com">百度</a>
      <li>Lorem.</li>
      <li>Expedita?</li>
      <li>Eveniet.</li>
      <li>Explicabo!</li>
      /* 最后一个li元素变红 */
      <li>Commodi.</li>
    </ul>
  </div>
</body>

```

4. last-of-type

```CSS
<style>
/* 选择li元素，并且是所有li元素中的最后一个 */
    li:last-of-type{
      color: red;
    }
</style>

<body>
  <div>
    <ul>
      <a href="www.baidu.com">百度</a>
      <li>Lorem.</li>
      <li>Expedita?</li>
      <li>Eveniet.</li>
      <li>Explicabo!</li>
      /* 最后一个li元素变红 */
      <li>Commodi.</li>
    </ul>
  </div>
</body>

```

5. nth-child

选中指定的第几个子元素

even: 关键字，等同于2n，选择偶数元素

odd: 关键字，等同于2n+1 选择奇数元素

```html
<style>
    /* 必须是a元素，必须是第五个子元素。存在其他元素也会计算数量 */
    /* 参数可以为一个变量，如2n则是必须是li元素，并且必须是第偶数个子元素 */
    li:nth-child(5) {
      color: red;
    }
</style>

<body>
  <div>
    <ul>
      <a href="www.baidu.com">百度</a>
      <li>Lorem.</li>
      <li>Expedita?</li>
      <li>Eveniet.</li>
      <li>Explicabo!</li>
      <li>Commodi.</li>
    </ul>
  </div>
</body>

```

6. nth-of-type

选中指定的子元素中第几个某类型的元素

```html
<style>
    li:nth-of-type(even) {
      /* 不顾其他元素，只根据指定元素的个数进行设置 */
      color: red;
    }
</style>

<body>
  <div>
    <ul>
      <p>p元素</p>
      <li>Lorem.</li>
      <li>Expedita?</li>
      <p>p元素</p>
      <li>Eveniet.</li>
      <li>Explicabo!</li>
      <p>p元素</p>
      <li>Commodi.</li>
    </ul>
  </div>
</body>

```

## 更多伪元素选择器

1. first-letter

选中元素中第一个字母（文字）

```html

<style>
    p::first-letter{
        color:  red;
        font-size:  2em;
        font-weight: hold; 
    }
</style>

<body>
    <p>
        Lorem ipsum dolor sit amet consectetur, adipisicing elit. Illo minima labore deleniti eveniet eos? Tempora vero, ipsam, earum nesciunt similique saepe alias ducimus omnis eos explicabo quasi accusamus unde voluptates?
    </p>
</body>

```
