# 行盒的盒模型

常见的行盒:包含具体内容的元素span, strong, em, i, img、 video, audio

## 显著特点

1. 盒子沿着内容延伸，内容有多长行盒就会延伸多长。如果有下一个行盒会接这这个行盒的尾部生成。
2. 行盒不能设置宽高,调整行盒的宽高，应该使用字体大小、行高、字体类型，间接调整。

3. 内边距(填充区)：水平方向有效，垂直方向只会显示背景颜色，不会实际占据空间。

4. 边框：水平方向有效，垂直方向只会显示背景颜色，不会实际占据空间。

5. 外边距：边框：水平方向有效，垂直方向不会实际占据空间。

## 行块盒

**display:inline-block**  的盒子

1. 不独占一行
2. 盒模型中的所有尺寸都有效

## 空白折叠 
空白折叠，发生在行盒(行块盒)内部  或  行盒(行块盒)之间，会出现空隙

![空白折叠.png](https://raw.githubusercontent.com/baobao129/2025-blog-public/main/images/1770300340299_l8zu17.png)

```html
<div>
  <a href="">1</a>
  <a href="">2</a>
  <a href="">3</a>
  <a href="">4</a>
  <a href="">5</a>
  <a href="">6</a>
  <a href="">7</a>
</div>
```

比如这样的a元素之间就会生成空白间隙，即发生了空白折叠，无法进行解决，只能通过以下方式进行解决

```html
<div>
  <a href="">1</a><a href="">2</a><a href="">3</a><a href="">4</a><a href="">5</a><a href="">6</a><a href="">7</a>
</div>
```

这样才能消除空白间隙

## 可替换元素  和  非可替换元素

大部分元素，页面上显示的结果，取决于元素内容，称之为**非可替换元素**

少部分元素，页面上显示的结果，取决于元素属性，称之为**可替换元素**

可替换元素：img、video、audio

绝大部分可替换元素均为行盒。

可替换元素类似于行块盒，盒模型中所有尺寸都有效。

属性**object-fit**可以用于图片和视频，表示图片和视频的适应方式。有以下几种常用取值：
- contain:表示不打破图片和视频的比例，保证其图片不溢出，必须显示完全在元素中。
- fill:表示完全填充满整个元素中，但是无法保证图片和视频的比例。
- cover:表示完全填充满整个元素，并且不打破图片和视频的比例，但是会有可能溢出元素，无法显示完全
